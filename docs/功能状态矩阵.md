# 功能状态矩阵

更新时间：2026-05-02

本矩阵用于约束文档声明，避免把占位、模拟或实验性能力描述为正式发布能力。状态定义如下：

| 状态 | 含义 | 发布说明要求 |
| --- | --- | --- |
| Beta | 有软件主链路和当前自动化门禁支撑，但仍缺发布 smoke 或真实设备证据 | 可以写入当前 Beta 功能说明，必须保留限制 |
| fixture | 有 fixture、golden、离线回放或自动化测试证据 | 可以写入测试覆盖说明，不得写成真实设备能力 |
| framework | 有适配框架、接口或检测口径，但缺具体设备记录 | 必须标注设备记录 pending |
| experimental | 有主链路或框架，但缺少真实设备、真实样本或跨平台验证 | 必须标注 experimental 和限制 |
| pending | 尚未接入主链路，或需要真实设备/发布 smoke 证据后才能提升 | 只能出现在路线图、风险或待办 |
