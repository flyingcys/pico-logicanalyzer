## 二、本轮已完成的改进

### 2.1 清除 webview 死代码（53%→59%，分母修正）
- 背景：项目有两套前端代码 `src/webview/`（旧）与 `src/frontend/`（新）。核查确认插件实际加载 `src/frontend`（webpack 入口 = `src/frontend/app/main-vscode.ts`，`out/webview` 仅是输出目录名）。
- 证据：全仓零源码 import `src/webview`；GitNexus `webview/utils/LayoutManager` 上游 0 调用；`webview/engines/` 9 个渲染引擎（~5000 行）孤儿闭环，不进 bundle。
- 操作：删 `src/webview/`（27 文件）+ `tests/unit/webview/LayoutManager.resource.test.ts`；`tsconfig.json` 移除 shims 引用；保留 webpack `webviewConfig`（frontend 构建链）。

## 五、剩余缺口与优先级

### driver-sdk（整体 8.5%，已判定内部脚手架）
GitNexus 确认：`src/` 零文件 import driver-sdk；6 个真实驱动全部绕过 SDK 直接用 `AnalyzerDriverBase`；5 个核心符号（DriverValidator/ProtocolHelper/HardwareCapabilityBuilder/DriverTester/DriverUtils）外部运行时调用者 = 0。
- 定位：**内部脚手架**（非第三方战略 SDK），不设门禁，不重点补单测。
- 若未来转战略定位，按 `docs/driver-sdk-定位与测试策略.md` 第五节清单补 tools 层。
- 详见 `docs/driver-sdk-定位与测试策略.md`、`docs/覆盖率门禁配置建议.md`。
