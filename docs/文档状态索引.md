## 更新规则

- 新增能力必须同步更新功能状态矩阵或专题文档。
- 发布说明不得使用绝对完成度、已认证支持、无条件兼容等措辞，除非 release gate 和认证矩阵有证据。
- SDK 模板和自动生成文档必须标注 `experimental`、模板待实现项，以及是否依赖 mock、fixture 或真实硬件。
- HTML/PDF/ZIP/项目包导出在未接入真实生成链路前，只能写作实验性或规划能力。
- 历史分析文档保留原始内容时，必须在索引中标注状态，避免与当前口径混用。
