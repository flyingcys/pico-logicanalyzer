## 更新规则

- 新增能力必须同步更新功能状态矩阵或专题文档。
- 发布说明不得使用绝对完成度、无条件兼容或真实设备通过结论等措辞，除非当前状态基线、release gate 和硬件矩阵均有证据。
- SDK 模板和自动生成文档必须标注 `experimental`、模板待实现项，以及是否依赖 mock、fixture 或真实设备。
- HTML/PDF/ZIP/项目包导出在未接入真实生成链路前，只能写作实验性或规划能力。
- 历史分析文档保留原始内容时，必须在索引中标注状态，避免与当前口径混用。
- 引用历史评审、历史计划或追加完成记录时，必须同时核对当前有效矩阵和 release gate；历史文档中的旧绝对完成度、生产可用、无条件兼容或设备通过表述不得单独作为当前状态证据。
