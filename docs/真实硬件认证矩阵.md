## 自动化验证路径

| 类别 | 命令或路径 | 验证内容 |
| --- | --- | --- |
| USB Pico 识别 | `npm exec -- jest tests/unit/drivers/HardwareDriverManager.business.test.ts --runInBand` | VID/PID `1209/3020` 精准识别，普通 Pico 降级为待握手候选 |
| 网络发现口径 | `NetworkDetector.getDefaultPorts()` 单元测试 | 统一端口为 `4045/10429/5555/5025/111` |
| Pico TCP 候选 | `NetworkDetector.fromProbeResult()` 单元测试 | 版本握手通过为高置信设备，端口开放仅为待验证候选 |
| 认证矩阵口径 | 本文档 + `docs/worktrees/05-device-discovery-certification.md` | 明确区分 framework、fixture、candidate、experimental |

## 发现状态口径

| Profile | 候选条件 | 已确认条件 | 自动化证据 |
| --- | --- | --- | --- |
| Pico W TCP | `4045` 端口开放或 UDP 广播响应 | Pico 版本握手返回有效版本、`FREQ`、`BLASTFREQ`、`BUFFER`、`CHANNELS` | `NetworkDetector.fromProbeResult()`、`WiFiDeviceDiscovery.parseDeviceResponse()` |
| Saleae Logic 2 API | `10429` 端口开放 | 后续接入 Saleae API 枚举并返回设备清单 | 当前只允许候选状态，不写成真实硬件认证 |
| Rigol/Siglent SCPI | `5555/5025/111` 端口开放 | `*IDN?` 返回包含 Rigol 或 Siglent | `NetworkDetector` 的 SCPI profile 握手路径 |
| sigrok-cli | `sigrok-cli --scan` 可执行 | 扫描输出列出具体设备和连接串 | `SigrokDetector` 输出作为外部依赖证据，仍需设备级认证 |

## 认证记录模板

每次真实硬件认证必须追加以下信息：

| 字段 | 内容 |
| --- | --- |
| 日期 | YYYY-MM-DD |
| 操作系统 | Windows / macOS / Linux 版本 |
| 设备型号和序列号 | 真实型号、固件版本、序列号可脱敏 |
| 扩展版本和 commit | VSIX 版本、Git commit |
| 命令或操作路径 | VSCode 命令、CLI 命令或测试脚本 |
| 采集配置 | 频率、pre/post、通道、触发 |
| 结果文件 | `.lac`、CSV、日志、截图路径 |
| 结论 | 通过 / 失败 / 阻塞，并链接问题 |
